* любой Java проект состоит из множества ОБЪЕКТОВ, которые между собой взаимодействуют (как в жизни)
* каждый объект обычно имеет: СВОЙСТВА (поля) и МЕТОДЫ (действия) – это основные рабочие механизмы
* объект в коде создается ключевым слово new
* класс (class) – это трафарет, слепок для создания объектов (экземпляр класса, instance) уже с конкретными данными (полями)
* класс и объект – при описания кода это одно и то же (хотя правильней говорить, что класс находится в коде во время разработки, а объект – в памяти во время выполнения программы)
* на основе одного класса можно создавать любое количество объектов (вы сами в коде управляете, какие именно объекты создавать)
* классы нужно сразу правильно распределять по пакетам (группировка по смыслу)
* пакеты именуются по доменному принципу от общего к частному (ru.javabegin.oop)
* для доступа одного класса к другому, если они находятся в разных пакетах, нужно выполнять import
*если классы находятся в одном пакете, они автоматически “видят” друг друга
* в одном файле .java может находиться только один public class, название которого должно совпадать с названием файла
* весь основной код пишется внутри класса (фигурных скобок)
* всегда следите за фигурными скобками, которые должны открываться и закрываться (обозначают границы класса, метода)
* не забываем в конце каждой строки прописывать точку с запятой “;” иначе компилятор не сможет разделить правильно ваш код (примечание: в Kotlin точка с запятой не требуется, только в редких случаях)
* закомментированные строки не выполняются (они просто нужны для информации)
